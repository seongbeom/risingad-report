# 광고채널 자동화 — 자격증명 발급 요청서 (보강판)

작성 기준일: 2026-06-15 · 출처: 각 플랫폼 공식 개발자 문서/공지 (본문 하단 출처)

> **사용법**: 채널별 **"📋 요청 문구"** 박스를 그대로 복사해 계정주/대행사에 전달.
> **"🔧 우리가 직접 함"** 항목은 우리가 앱·토큰을 만드는 부분이라 요청 불필요.
> 받은 값은 `/admin`(계정관리)에서 **매장마다** 입력 → `accounts` 테이블에 저장(메타/네이버와 동일).

---

## 0. 한눈에 보기 — "광고주 개별 계정으로 받을 수 있나?" (= 네이버 검색광고처럼)

가장 중요한 기준. **검색광고처럼 광고주별로 키를 받으면 되는지**, 아니면 공식대행사/특수 경로가 필요한지.

| 채널 | 개별 광고주 발급 | 게이트(걸림돌) | 검색광고 대비 | 자동화 방식 |
|---|---|---|---|---|
| **네이버 검색광고** | ✅ YES | 없음 (자기 계정서 즉시 발급) | 기준 | API |
| **메타(FB/IG)** | ✅ YES | 없음 (본인 소유 계정은 앱심사 면제) | 거의 같음 | API |
| **크리테오** | 🟡 조건부 | Partners Portal 접근을 Criteo 담당자가 열어줘야 함 | 더 번거로움 | API |
| **구글** | 🟡 조건부 | **MCC(관리자계정) 필수** + Basic Access 승인 | 더 번거로움 | API |
| **틱톡** | 🟡 조건부 | **앱 심사 필수**(자기 계정도) 2~7영업일 | 더 번거로움 | API |
| **카카오모먼트**(DA/모객/메시지) | 🟡 조건부 | 광고주 개별 신청 가능하나 **카카오 수동 승인** | 더 번거로움 | API |
| **네이버 GFA(성과형 DA)** | 🔴 **NO** | **공식파트너사/대표관리계정 한정** | 광고주 개별 불가 | 크롤러/대행사 |
| 쇼핑박스 PC·MO, 다음쇼핑박스, 네이트 | 🔴 NO (공식 API 없음) | — | — | 크롤러/리포트 |

**결론**: **GFA 하나 빼고는 전부 "광고주 개별 계정"으로 받을 수 있음**(난이도 차이만 존재). 메타가 검색광고급으로 제일 쉽고, 구글·틱톡·크리테오·카카오는 광고주 개별 경로가 있으나 각자 승인 게이트 하나씩 있음. **GFA만 광고주 개별 불가 → 공식대행사 경유 필요.**

## 우리 시스템이 자격증명을 받는 방식 (참고)

| 채널 | 저장 위치 | 인증 모델 |
|---|---|---|
| 검색광고 | `accounts.naver_api_key`/`naver_secret`/`naver_customer_id` | **매장마다 키 세트** (계정마다 발급) |
| 메타 | `accounts.meta_account_id` + 서버 `META_ACCESS_TOKEN` | **토큰 1개 + 매장별 계정ID** |
| 신규(GFA/카카오 등) | `accounts`에 컬럼 추가 예정 | 아래 채널별 참고 |

> 인증 모델 2가지: **(A) 매장마다 키 따로**(검색광고·크리테오) / **(B) 앱·토큰 1개 + 매장은 계정ID로 구분**(메타·GFA·카카오·틱톡). B형은 앱 1개만 만들면 매장 수와 무관.

# A. API 연동 채널 (공식 API 있음)

## A-1. 네이버 검색광고 ✅ 개별 즉시 — 나머지 매장만

가장 쉬움. 매장 마스터가 화면에서 3개 값 복사하면 끝. 심사 없음.

**🔧 우리가 직접 함**: HMAC 서명 호출(구현 완료)

**📋 요청 문구**
```
네이버 검색광고 API 키 발급 부탁드립니다.
1) searchad.naver.com 로그인 → 광고시스템 진입
2) [도구] > [API 사용 관리]
3) 아래 3개 복사 전달:
   - 액세스라이선스(API_KEY)
   - 비밀키(SECRET_KEY)
   - CUSTOMER_ID (같은 화면의 광고주 번호)
※ 매장(광고계정)마다 별도 발급. 마스터 권한에서 발급해 주세요.
```
**대안**: 우리 대행사 계정에 매장이 멤버 권한 부여 시, 우리 키 하나 + 매장 CUSTOMER_ID로도 조회 가능.

---

## A-2. 메타 (FB/IG) ✅ 개별 가능 — 미연결 매장만

토큰 발급 완료. 매장 광고계정 ID만 받으면 됨. **본인 소유 계정은 앱심사 불필요.**

**📋 요청 문구**
```
Meta(페이스북/인스타) 광고계정 ID를 알려주세요.
- 광고관리자 URL의 act_ 뒤 숫자 (예: act_1234567890 → 1234567890)
- 저희 비즈니스 자산(시스템 사용자)에 해당 광고계정 '분석/리포트' 권한 부여
```

---

## A-3. 크리테오 🟡 개별 가능(포털 접근만 열리면) — 앱심사 없음

광고주가 Partners Portal에서 셀프로 앱 생성·키 발급·consent까지 가능. **단 포털 로그인 권한이 없으면 Criteo 담당자에게 계정 개설 요청 필요**(자유가입 아님). 토큰은 15분짜리, 우리가 자동 갱신.

**📋 요청 문구**
```
Criteo Marketing Solutions API 자격증명 발급 부탁드립니다. (admin/business manager 권한)
※ Criteo Partners Portal 로그인 권한이 없으면, Criteo 계정 담당자에게
  포털 접근 계정 개설을 먼저 요청해 주세요.
1) Partners Portal > My apps > 앱 생성
2) 인증방식 'Client Credentials' 선택
3) 서비스 'C-Growth (Marketing Solutions)' 선택  ※ C-Max(Retail Media) 아님
4) 'Create new key' → client_id / client_secret 전달 (이때만 보임, 분실 시 재발급)
5) consent URL 생성 → 전달, 대상 advertiser_id 포함
```

---

## A-4. 구글 광고 🟡 개별 가능하나 MCC 필수

핵심 게이트: **developer token은 MCC(관리자 계정)에서만 발급**. 일반 광고계정만으론 불가. 광고주가 직접 MCC를 만들 수는 있으나, 검색광고처럼 자기 계정 화면에서 바로 키가 안 나옴. 그래서 보통 **우리(대행사) MCC로 처리**가 깔끔.

**🔧 우리가 직접 함**
- Developer Token (우리 MCC `ads.google.com/aw/apicenter`, **Basic access 신청 ~5영업일**, 실데이터엔 Basic 이상 필수)
- Google Cloud 프로젝트 + OAuth client_id/secret + refresh_token

**📋 요청 문구**
```
구글 광고 데이터 연동을 위해 둘 중 하나 부탁드립니다.
A) (권장) 우리 대행사 MCC(관리자계정 ID: ___-___-____)로 광고계정 연결 수락
   + 조회 대상 customer_id(광고계정 ID, 10자리) 전달
B) 또는 우리가 보내는 OAuth 동의 링크에 광고주 계정으로 1회 로그인·동의
   + customer_id 전달
```

---

## A-5. 틱톡 🟡 개별 가능하나 앱 심사 필수

광고주 본인 등록("Direct Advertiser") 경로는 있으나, **자기 계정 실데이터 조회도 앱 심사(2~7영업일)** 통과 필요. 게이트가 강해 일정 여유 필요.

**🔧 우리가 직접 함**
- TikTok for Business Developer 등록 → 앱 생성(App ID/Secret) → **앱 심사 통과(~7영업일)**
- OAuth로 access_token 교환(장기 토큰)

**📋 요청 문구**
```
틱톡 광고 데이터 연동을 위해:
1) advertiser_id(Ads Manager의 광고계정 숫자 ID) 전달
2) 우리가 보내는 인가 링크에 본인 Ads Manager 계정으로 1회 로그인·동의
```

## A-6. 카카오 (DA / 모객 / 메시지) 🟡 개별 가능(카카오 승인 필요)

**중요 정정**: 과거 "공식대행사 한정"이었으나 **현재는 "공식대행사 + 사전 협의된 광고주"** 로 완화됨. 즉 **광고주 개별 신청 가능**. 단 네이버 검색광고처럼 즉시 셀프가 아니라 **카카오 수동 승인(영업일 3~5일)** 필요. **DA·모객·메시지는 카카오모먼트 API 하나로 다 됨**(채널별 별도 신청 X).

**인증 모델 = 멤버십형**(네이버 GFA의 위임 계층형과 다름): 토큰을 발급하는 **카카오계정이 그 광고계정(adAccountId)의 마스터/멤버**여야 호출 가능. 공식대행사 권한을 "빌려쓰는" 게 아니라 **내 계정이 그 광고계정 멤버로 등록**돼야 함. 대행사는 **영업권 수락** 또는 멤버 등록으로 멤버 권한 확보 후, adAccountId를 바꿔가며 다수 계정을 한 앱으로 호출.

**🔧 우리가 직접 함**: Kakao Developers 앱(REST/Admin 키), 비즈앱 전환+본인인증, OAuth 토큰(access 6h/refresh 2개월) 자동 갱신

**📋 요청 문구 (광고주 개별 경로)**
```
카카오 광고(디스플레이/모객/메시지) 데이터를 카카오모먼트 API로 연동하려 합니다.
('DA/모객/메시지'는 카카오모먼트 API 하나로 다 처리됩니다)

[확인] 대상 카카오모먼트 광고계정(adAccountId)과,
       그 광고계정에 마스터/멤버 권한을 줄 수 있는지 확인 부탁드려요.
[필요] 1) 저희가 카카오모먼트 API 권한 신청을 진행합니다
          (담당 카카오 마케터 통해, 또는 카카오모먼트 공지 구글폼으로 — 승인 3~5영업일)
       2) 저희(또는 대행사) 카카오계정을 해당 광고계정의 멤버로 등록
          + 카카오비즈니스 약관 동의
       3) 대상 adAccountId 전달
```

> **공식대행사 경유가 필요한 경우**: 우리가 직접 권한 승인을 못 받으면, 광고주를 관리하는 공식대행사가 자기 앱으로 호출해 데이터를 전달하는 방식도 가능.

# B. 크롤러/대행사 채널 (광고주 개별 발급 불가)

## B-1. 네이버 GFA (성과형 디스플레이) 🔴 광고주 개별 불가 — 공식대행사 필요

**유일하게 광고주 개별 발급이 막힌 채널.** 공식명 "네이버 성과형 디스플레이 광고 API(Beta)". 베타라 **공식 파트너사로 승인된 "대표 관리 계정"만** API 사용 신청 가능. 검색광고와 **완전 별개 시스템**(검색광고 키 재사용 불가).

**권한 구조 = 위임 계층형**
```
공식대행사 대표 관리 계정   ← GFA API 권한(파트너 승인) + 앱(Client ID) 등록은 여기서만
   └─ 우리 재대행사 관리 계정   ← 우리가 로그인/권한 갖는 레벨
        ├─ 광고주A (매장1)   ← 우리 밑에 묶여 있으면
        ├─ 광고주B (매장2)      한 번에 다 호출됨
        └─ 광고주C (매장3)
```

- API로 **노출·클릭·광고비·전환·매출 전부 조회 가능**(`impCount`/`clickCount`/`convCount`/`sales` 등). 권한만 풀리면 데이터는 다 나옴.
- **재대행사는 직접 신청 불가.** 상위 공식대행사가 권한 보유해야 하고, 우리 매장이 그 하위 계층에 묶여 있으면 우리 앱으로 호출.
- 인증: 네이버 로그인 OAuth + Client ID. 앱(Client ID)은 우리가 우리 계정으로 1개 만들면 되나, **API 사용 신청(등록·심사)은 대표 관리 계정에서만** 가능.

**진행 방식 (둘 다 충족 시)**
1. 우리가 네이버 개발자센터에서 앱 생성 → Client ID 전달 (매장 수 무관 1개)
2. 공식대행사가 그 Client ID를 대표 관리 계정 [설정 > API 관리 > API 사용 신청]에 등록·승인
3. 우리 매장은 이미 재대행사 하위라 → 승인되면 우리 쪽에서 자동 수집 시작

**📋 재대행사에 보낼 문구**
```
GFA(성과형 디스플레이광고) 데이터를 API로 자동 연동하려 합니다.
아래 2가지 확인 부탁드리고, 둘 다 되면 이어서 처리 부탁드려요.

[확인 1] 저희가 관리하는 광고주(매장)들이 재대행사 계정 하위에 묶여 있나요?
[확인 2] 상위 공식대행사 대표 관리 계정이 네이버 '성과형 디스플레이 광고 API(Beta)'
        사용 승인을 받았나요? (아직이면, 신청 가능한지도요)

[둘 다 YES면 → 진행]
 1) 저희가 연동용 앱을 만들어 Client ID를 전달드립니다 (매장 수 무관, 1개)
 2) 공식대행사가 그 Client ID를 대표 관리 계정
    [설정 > API 관리 > API 사용 신청]에 등록·승인
 → 저희 매장들은 이미 재대행사 계정 하위에 있으니,
   승인되면 저희 쪽에서 바로 자동 수집을 시작합니다.

[확인 2가 NO면 → 대안]
 - 공식대행사가 GFA 광고관리자에서 저희 매장 성과 리포트를
   일/주 단위 엑셀·CSV로 정기 전달해 주실 수 있을까요?
```

## B-2. 공식 API 없는 채널 — 크롤러/리포트

| 채널 | 공식 API | 받아야 할 것 | 걸림돌 |
|---|---|---|---|
| 쇼핑박스 PC | ✗ (보장형) | 리셀러(NHN커머스 등) 정산 리포트, 관리자 계정 | 데이터가 리셀러 리포트 중심 → 파일 수령+파싱 권장 |
| 쇼핑박스 MO(트렌드픽) | ✗ (보장형) | 동일 | 동일 |
| 다음 쇼핑박스 | ✗ (카카오 API 명시 제외) | 카카오비즈니스 로그인, 광고계정 멤버권한 | 관리자 화면 크롤링 외 대안 없음 |
| 네이트 CPC | ✗ | 네이트 광고관리자 ID/PW, 사업자정보 | 캡차·세션 짧음, 리포트 다운로드 자동화가 현실적 |

**크롤러 공통 요청 문구**
```
(공식 API가 없어 관리자 페이지 자동수집/리포트 수령이 필요합니다)
1) 광고 관리자 로그인 ID / 비밀번호
2) 2차인증 방식(SMS/OTP/카카오앱) 및 인증 시 연락 가능한 담당자
3) 고정 서버 IP를 신뢰 기기로 등록 가능한지
4) (보장형) 리셀러 정산 리포트 정기 수신 경로(이메일 등)
※ 로그인 정보는 평문 전달 금지 — 암호화 채널 또는 권한 분리된 서브계정 권장
```

# 우선순위 (난이도 vs 효과)

1. **네이버 검색광고 나머지 매장** — 즉시, 광고주 부담 최소(값 3개 복사)
2. **메타 미연결 매장** — 즉시, 광고계정 ID + 권한
3. **크리테오** — 포털 접근만 열리면 셀프, 앱심사 없음
4. **구글** — 우리 MCC/Basic 승인(~5일) 선행, 광고주는 연결만
5. **틱톡** — 앱 심사(~1~2주) 감안해 일찍 착수
6. **카카오** — 광고주 개별 신청 가능(카카오 승인 3~5일), DA/모객/메시지 한 번에
7. **네이버 GFA** — 공식대행사 권한 확인이 선행(막히면 리포트 전달 폴백)
8. **크롤러 채널(쇼핑박스/다음/네이트)** — 계정·2FA 협의 + 서버 IP 신뢰등록 후 개별 구축
