# 신규 광고채널 연동 — 광고주(일반 유저)에게 요청할 것

작성 기준일: 2026-06-15
대상: **아직 연동 안 한 채널만** (메타·네이버 검색광고는 완료 → 제외)

> **원칙**: 요청 상대는 일반 유저(광고주)라 "키 발급"은 어려워함.
> → **1순위 = 계정 권한만 요청**(우리를 멤버로 추가 / 연동 동의 클릭 — 쉬움).
> 키·토큰은 **우리가 직접 생성**. 광고주는 "계정 열어주기"만.
> → **계정 권한이 어려우면 2순위 = 키 발급 요청**으로 폴백.

---

## 핵심 — 거의 모든 채널이 "계정 권한"으로 됨

우리가 각 플랫폼에 **우리 앱/개발자계정/MCC를 미리 만들어 두면**, 광고주는 키를 만질 필요 없이 **"우리한테 자기 광고계정 권한을 주거나 동의 버튼만"** 누르면 돼요. 키는 우리가 그 권한으로 생성/조회.

| 채널 | 1순위 (광고주: 쉬움) | 2순위 (어려우면) | 우리가 미리 할 것 |
|---|---|---|---|
| 크리테오 | 우리 consent 링크 동의 | client_id/secret 발급 | Criteo 파트너 앱 |
| 구글 | 우리 MCC에 계정 연결 수락 | customer_id만 알려주기 | MCC + Developer Token |
| 틱톡 | 우리 인가 링크 동의 | advertiser_id만 알려주기 | 앱 + 심사 |
| 카카오 | 우리를 광고계정 멤버 추가 | adAccountId만 알려주기 | 카카오 앱 + API권한 |
| 네이버 GFA | (공식대행사 처리) | — | Client ID |
| 쇼핑박스/다음/네이트 | 서브계정 로그인 위임 | 리포트 정기 전달 | 크롤러 |

---

## 1. 크리테오

**1순위 (계정 동의)**: 우리가 보낸 **consent(연동 동의) 링크를 광고주가 클릭·승인**.
→ 승인되면 우리가 그 광고주 데이터에 접근. 광고주는 키 안 만짐.
**2순위 (어려우면)**: 광고주가 Partners Portal에서 client_id/secret 발급해 전달.
**[우리]**: Criteo Partners Portal에 우리 파트너 앱 생성(없으면 Criteo 담당자에 포털 개설 요청)

**요청 문구**
```
크리테오 광고 데이터 연동하려 합니다.
제가 보내드리는 '연동 동의' 링크 한 번만 클릭해서 승인해 주시면 됩니다.
(별도로 발급하실 건 없습니다. 어떤 광고주 계정인지만 알려주세요.)
```

---

## 2. 구글 광고

**1순위 (계정 연결)**: 우리 **대행사 MCC(관리자계정)에 광고주 계정 연결 요청**을 보내면, 광고주가 **수락 버튼**만 누름. (또는 우리가 보낸 OAuth 동의 링크 1회 클릭)
**2순위 (어려우면)**: 광고주가 자기 **customer_id(광고계정 ID, 10자리)** 만 알려주기.
**[우리]**: MCC + Developer Token(Basic, ~5영업일), OAuth client/refresh token

**요청 문구**
```
구글 광고 데이터 연동하려 합니다.
제 관리자계정에서 '계정 연결 요청'을 보내드릴 테니, 받으시면 수락만 눌러주세요.
(또는 제가 보내는 동의 링크 1회 클릭)
연결되면 제가 데이터를 가져갑니다.
```

---

## 3. 틱톡

**1순위 (계정 인가)**: 우리가 보낸 **인가 링크를 광고주가 본인 Ads Manager 계정으로 1회 로그인·동의**.
→ 동의 시 우리가 advertiser_id 접근 확보. 키 안 만짐.
**2순위 (어려우면)**: 광고주가 **advertiser_id(광고계정 숫자 ID)** 알려주기.
**[우리]**: TikTok Developer 등록, 앱 생성, **앱 심사 통과(~7영업일)**, access token

**요청 문구**
```
틱톡 광고 데이터 연동하려 합니다.
제가 보내는 '연동 동의' 링크에 틱톡 광고계정으로 한 번 로그인·동의만 해주시면 됩니다.
```

---

## 4. 카카오 (DA / 모객 / 메시지 — 한 번에)

DA·모객·메시지 **한 연동으로 다 됨**.
**1순위 (멤버 추가)**: 광고주가 **카카오모먼트 광고계정에 우리 카카오계정을 멤버로 추가** (또는 우리 대행사 영업권 수락) + 카카오비즈니스 약관 동의(클릭).
→ 멤버가 되면 우리가 그 계정 데이터 조회. 키 안 만짐.
**2순위 (어려우면)**: 광고주가 **adAccountId(광고계정 ID)** 알려주기.
**[우리]**: Kakao Developers 앱, 비즈앱 전환+본인인증, **카카오모먼트 API 권한 신청(승인 3~5영업일)**, 토큰 자동 갱신

> 카카오는 우리 계정이 그 광고계정 멤버여야 호출됨(멤버십형). 그래서 "멤버 추가"가 1순위.

**요청 문구**
```
카카오 광고(디스플레이/모객/메시지) 데이터 연동하려 합니다.
카카오모먼트 광고계정 설정에서 제 카카오계정을 '멤버'로 추가해 주시고,
카카오비즈니스 약관 동의만 해주시면 됩니다.
(제 카카오계정 ID는 따로 알려드릴게요. 광고계정 이름만 알려주세요.)
```

---

## 5. 네이버 GFA (성과형 디스플레이) — 공식대행사 경유

광고주(일반 유저)에게 직접 받을 게 없음. **상위 공식대행사**가 처리.
**확인할 것** (재대행사/공식대행사 측):
- 우리 매장들이 재대행사 계정 하위에 묶여 있는지
- 공식대행사 대표 관리 계정이 GFA API 사용 승인 보유했는지
**[우리]**: Client ID 생성(1개) → 공식대행사가 대표 관리 계정에 등록·승인

**요청 문구 (재대행사/공식대행사용)**
```
GFA(성과형 디스플레이) 데이터 연동 확인 부탁드려요.
1) 저희 매장들이 재대행사 계정 하위에 묶여 있나요?
2) 공식대행사가 '성과형 디스플레이 광고 API(Beta)' 사용 승인을 받았나요?
   → YES면 저희가 Client ID 만들어 드릴 테니 대표 관리 계정 [설정>API관리]에 등록·승인 부탁드립니다.
   → 안 되면 광고관리자 리포트를 엑셀/CSV로 정기 전달 가능할까요?
```

## 6. 크롤러 채널 (쇼핑박스 PC·MO / 다음 쇼핑박스 / 네이트)

공식 API 없음.
**1순위**: 광고주가 **권한 분리된 서브계정(리포트 조회용) 로그인**을 만들어 위임.
**2순위**: 광고주 본인 로그인 ID/PW + 2차인증 방식(암호화 채널 전달).
**대안(보장형: 트렌드픽/다음쇼핑박스)**: 리셀러 정산 리포트 정기 전달.

**요청 문구**
```
(공식 연동이 없어 관리자 페이지 접근이 필요합니다)
1) 가능하면 리포트 조회용 서브계정을 하나 만들어 로그인 정보를 주세요.
2) 2차인증 방식(SMS/OTP/카카오앱)과 인증 시 연락 가능한 분도요.
(보장형 상품이면, 리셀러 리포트를 정기적으로 받는 방법도 가능합니다.)
```

---

## 정리

- **광고주에게는 "키 주세요" 대신 "권한 주세요 / 동의 눌러주세요"** — 크리테오·틱톡은 동의 링크 클릭, 구글은 계정 연결 수락, 카카오는 멤버 추가.
- **키·토큰·앱은 전부 우리가 생성.** 광고주가 만질 건 없음.
- 정 안 되면 그때만 "계정 ID 또는 키 발급" 폴백.
- **우리 선행작업**(MCC/Developer Token, 틱톡 앱심사, 카카오 API권한, 크리테오 파트너앱, GFA Client ID)은 광고주 응답과 **병행** 진행.
