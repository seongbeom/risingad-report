# 신규 광고채널 연동 — 거래처 요청 자료 목록

작성 기준일: 2026-06-15
대상: **아직 연동 안 한 채널만** (메타·네이버 검색광고는 연동 완료 → 제외)
용도: 거래처에서 아래 자료를 받으면 → 우리 시스템(계정관리 화면)에 매장별 입력 → 연동 진행

> 표기: **[값]** = 받아서 시스템에 입력할 자격증명/ID · **[액션]** = 거래처가 해줘야 하는 권한부여/동의 (값 아님)
> **[우리]** = 우리가 직접 만드는 부분이라 요청 불필요(참고용)

---

## 요청 난이도 한눈에

| 채널 | 거래처 부담 | 선결 조건 | 받는 핵심 값 |
|---|---|---|---|
| 크리테오 | 낮음 | 포털 접근권한 | client_id, client_secret, advertiser_id |
| 구글 | 낮음 | 우리 MCC 승인 | customer_id (+ 계정연결 수락) |
| 틱톡 | 낮음 | 우리 앱심사 | advertiser_id (+ 인가 동의) |
| 카카오 | 중간 | 카카오 API승인(3~5일) | adAccountId (+ 멤버권한·약관동의) |
| 네이버 GFA | 높음 | 공식대행사 권한 | (광고계정번호 + 공식대행사 처리) |
| 쇼핑박스/다음/네이트 | 높음 | 로그인 위임 | 관리자 ID/PW, 2차인증 |

---

## 1. 크리테오 (Criteo)

매장마다 받을 값 3개. 앱 심사 없음.

**받을 자료**
- **[값]** `client_id` — OAuth2 앱 식별자
- **[값]** `client_secret` — 앱 비밀키 (다운로드 시점에만 보임, 분실 시 재발급)
- **[값]** `advertiser_id` — 대상 광고주 계정 ID
- **[액션]** consent(권한 부여) — 우리가 보낸 consent URL을 광고주가 승인, 또는 광고주가 생성한 consent URL 전달

**거래처 발급 경로**: Criteo Partners Portal > My apps > 앱 생성(Client Credentials 방식, **C-Growth/Marketing Solutions** 선택) > Create new key
**선결**: Partners Portal 로그인 권한 없으면 Criteo 담당자에게 포털 계정 개설 먼저 요청
**[우리]**: 15분짜리 access token 자동 발급/갱신

**요청 문구**
```
Criteo API 연동 자료 부탁드립니다. (매장별)
1) client_id, client_secret  (Partners Portal > My apps > 앱 생성 >
   Client Credentials + C-Growth(Marketing Solutions) > Create new key)
2) advertiser_id (대상 광고주 계정 ID)
3) 저희가 보내는 consent URL 승인 (또는 consent URL 전달)
※ Partners Portal 접근이 없으면 Criteo 담당자에게 계정 개설을 먼저 요청해 주세요.
```

## 2. 구글 광고 (Google Ads)

매장마다 받을 건 customer_id 하나 + 계정 연결 수락. 나머지는 우리가 만듦.

**받을 자료**
- **[값]** `customer_id` — 광고계정 ID (10자리, 하이픈 제외)
- **[액션]** 둘 중 하나:
  - (권장) 우리 대행사 **MCC(관리자계정)로 광고계정 연결 수락**
  - 또는 우리가 보낸 **OAuth 동의 링크에 광고주 계정으로 1회 로그인·동의**

**[우리]**: Developer Token(우리 MCC, Basic access 승인 ~5영업일), Google Cloud OAuth client, refresh token

**요청 문구**
```
구글 광고 연동 자료 부탁드립니다. (매장별)
1) customer_id (광고계정 ID, 10자리)
2) 아래 중 하나:
   A) 우리 대행사 MCC(ID: ___-___-____)로 광고계정 연결 수락
   B) 우리가 보내는 OAuth 동의 링크에 광고주 계정으로 1회 동의
```

## 3. 틱톡 (TikTok)

매장마다 advertiser_id + 인가 동의. 앱·심사는 우리가.

**받을 자료**
- **[값]** `advertiser_id` — 광고계정 숫자 ID (Ads Manager에서 확인)
- **[액션]** 우리가 보낸 **인가(authorization) 링크에 본인 Ads Manager 계정으로 1회 로그인·동의**

**[우리]**: Developer 등록, 앱 생성(App ID/Secret), **앱 심사 통과(~7영업일)**, OAuth access token

**요청 문구**
```
틱톡 광고 연동 자료 부탁드립니다. (매장별)
1) advertiser_id (Ads Manager의 광고계정 숫자 ID)
2) 우리가 보내는 인가 링크에 본인 Ads Manager 계정으로 1회 동의
```

## 4. 카카오 (DA / 모객 / 메시지 = 카카오모먼트)

DA·모객·메시지 **한 API로 다 됨**. 매장마다 adAccountId + 멤버권한.

**받을 자료**
- **[값]** `adAccountId` — 카카오모먼트 광고계정 ID
- **[액션]** 우리(또는 대행사) 카카오계정을 **해당 광고계정의 마스터/멤버로 등록** (또는 대행사 영업권 수락)
- **[액션]** **카카오비즈니스 이용약관 동의**

**[우리]**: Kakao Developers 앱(REST/Admin 키), 비즈앱 전환+본인인증, **카카오모먼트 API 권한 신청(승인 3~5영업일)**, OAuth 토큰(access 6h/refresh 2개월) 자동 갱신

> 인증 모델 = 멤버십형. 우리 카카오계정이 그 광고계정 멤버여야 호출됨(권한 빌려쓰기 아님).

**요청 문구**
```
카카오 광고(DA/모객/메시지) 연동 자료 부탁드립니다.
(카카오모먼트 API 하나로 셋 다 처리됩니다 / 매장별)
1) adAccountId (카카오모먼트 광고계정 ID)
2) 저희 카카오계정을 해당 광고계정의 멤버로 등록 (또는 대행사 영업권 수락)
3) 카카오비즈니스 이용약관 동의
※ 카카오모먼트 API 사용 권한은 저희가 신청 진행합니다(카카오 승인 3~5영업일).
```

## 5. 네이버 GFA (성과형 디스플레이) — 공식대행사 경유 필수

광고주 개별 발급 불가. 상위 공식대행사가 처리해줘야 함.

**확인/받을 자료**
- **[확인]** 우리 매장들이 **재대행사 계정 하위에 묶여 있는지**
- **[확인]** 상위 **공식대행사 대표 관리 계정이 GFA API 사용 승인** 보유 여부 (없으면 신청 가능한지)
- **[액션]** 공식대행사가 **우리 Client ID를 대표 관리 계정 [설정>API관리>API 사용 신청]에 등록·승인**
- **[값]** 매장별 광고계정 번호 (호출 대상 지정용)

**[우리]**: 네이버 개발자센터 앱 생성(Client ID, 매장 수 무관 1개), OAuth 토큰

**요청 문구**
```
GFA(성과형 디스플레이) 연동 확인 부탁드립니다.
[확인 1] 저희 광고주(매장)들이 재대행사 계정 하위에 묶여 있나요?
[확인 2] 상위 공식대행사 대표 관리 계정이 네이버 '성과형 디스플레이 광고 API(Beta)'
        사용 승인을 받았나요? (없으면 신청 가능한지)
[YES면] 저희가 Client ID(1개)를 만들어 드릴 테니, 공식대행사 대표 관리 계정
        [설정>API관리>API 사용 신청]에 등록·승인 부탁드립니다.
[NO면]  GFA 광고관리자 성과 리포트를 일/주 단위 엑셀·CSV로 정기 전달 가능할까요?
```

## 6. 크롤러 채널 (쇼핑박스 PC·MO / 다음 쇼핑박스 / 네이트 CPC)

공식 API 없음. 로그인 위임 또는 리포트 수령.

**받을 자료**
- **[값]** 광고 관리자 **로그인 ID / 비밀번호** (암호화 채널 전달, 가능하면 권한분리 서브계정)
- **[값]** **2차인증 방식**(SMS/OTP/카카오앱) + 인증 시 연락 담당자
- **[액션]** 우리 **고정 서버 IP를 신뢰 기기로 등록** 가능 여부
- **[값/대안]** (보장형: 트렌드픽/다음쇼핑박스) **리셀러 정산 리포트 정기 수신 경로**(이메일 등)

**요청 문구**
```
(공식 API가 없어 관리자 페이지 자동수집/리포트 수령이 필요합니다 / 매체별)
1) 광고 관리자 로그인 ID / 비밀번호 (암호화 채널로)
2) 2차인증 방식(SMS/OTP/카카오앱) + 인증 시 연락 담당자
3) 고정 서버 IP를 신뢰 기기로 등록 가능한지
4) (보장형 상품) 리셀러 정산 리포트 정기 수신 경로
```

## 우리 쪽 진행 순서 (자료 받은 뒤)

1. 받은 값 → `accounts` 테이블 컬럼 추가 + 계정관리 화면 입력란 추가 (채널별)
2. 채널 API 모듈 작성(인증·수집) → 효율시트 칼럼 매핑
3. 수집 잡 스케줄 등록 (해당 채널 자격 설정된 매장만 대상)

> 선행 작업(우리 몫): 구글 Developer Token(Basic) 신청, 틱톡 앱 심사, 카카오모먼트 API 권한 신청, GFA Client ID 생성 — 거래처 자료와 병행 가능.
